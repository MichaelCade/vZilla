Lab Time... Finally

16/07/2013

2 Comments

 
So i have finally found some time to get back onto the home lab kit. A couple of things changed from the original plan, my plan was to add the netapp sims into the environment in an automated way.... this is not possible due to the sims only being available to netapp partners. So i was unable to do this, however i have added them into my lab, typically i had added these a few months back so they are version 8.1.2 and then a couple of weeks ago 8.2 has now been released. 

But anyhow, i now have 4 x NetApp Clustered OnTap Sims running inside my home lab as well as 2 x NetApp 7Mode Sims. I have not completed all the configuration on these sims, the time consuming thing seems to be around the sparse files that are shipped when you download the sims and they need to be manually expanded to work on ESX, i am not sure if this has been resolved with the latest downloads, i am yet to check. 

I have also created a brand new VMware View 5.2 environment in the lab, this was following the ICM course i attended a couple of weeks back, really good course, really good instructer and helped that i had seen and used View a bit in the past. So for this environment i have created the following. 

A View Connection Server joined to the lab.local domain 
A View Security Server sat in the DMZ well actually the Home network vswitch but simulating a DMZ in this instance. 
Some XP machines fully patched 
Some Windows 7 machines fully patched 
A Windows 8 machine fully patched to 8.1 
A Windows XP machine provisioned for ThinApp installations with clean snapshot taken. 
A Windows 7 machine provisioned for ThinApp installations with clean snapshot taken. 

i have briefly ran through the same labs that were present in the ICM course and proved everything is up and running which will provide with a good demo environment should i need that. 

So after getting little bits of lab time i thought about a few things that would also be good to get in the lab, apart from the obvious of the latest netapp sims and things like vcloud director among others, i thought along the lines to the other server i have sat in the office. and thought how i could utilize this to configure an SRM environment. so i made sure i had the downloaded kit from autolab (was going to make my life easier by using that on the second site) however i hit a wall and the ESX version just kept failing. cant remember if i had this problem before but its something i am going to have to look back over to get sorted. 

For the SRM aspect to work i am going to need to either add a physical card to the current lab_local vswitch on my live site or use the same configuration on the second site with the virtual router and basically configure both sites the same. Its something i am going to have to debate i believe i do have a spare nic on each server but would rather hold fire on using them just yet. 

On another side note i have picked up some Cisco kit, a couple of switches and a couple routers. just another little training/project on the list. 

The reason for the lack of communication on the new site is also due to 1, i got married 2, we bought a new house and 3, things are massively busy at work at the moment. i am trying to build an agenda of things to work through and implement into the lab environment and need to train myself to document my steps to share with you guys. 
if you have any questions then let me know, also anything you would like to see in upcoming posts.

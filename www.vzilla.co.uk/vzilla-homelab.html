HomeLab

This page describes what I have configured in my home lab. I have recently changed this from my previous due to slowness, I was using the AutoLab vApp which is a great tool if you just want an environment standing up very quickly, however I found it to limit my focus on new products and software because of all that automation, so I have taken the plunge once again and I have started to build a completely new lab environment, I am still using the same hardware. 

Currently my HomeLab has been a little neglected of late and is in need of a real bit of tender loving care. It looks as though the two SATA hard drives have a lot of bad sectors, not a huge problem as there is no mission critical data on them so it’s just a case of replacing them. I am in the process of seeing where best to upgrade the system and if I can in fact add an SSD because disk performance does make a huge difference on this test bed. As soon as I have this changed then I will update the content below to reflect.

Ideally I would like to upgrade the current setup that I have but also add a NAS device into the equation to allow for home media sharing around the house. This all comes down to funds though and at the moment I don't have them and I really do not have the time to get on and use the lab to its full potential.

My first ESXi host as shown below will have VMware vSphere 5.5 installed, this system will act as my live site, and this server generally runs 24x7 as this also houses my home desktop machine as a virtual machine. My intention here is to have a lightweight virtual router running separating the home network and lab network. I will then have a 2012 R2 Domain Controller as well as a VMware vSphere 5.5 Virtual Centre. These three virtual machines will be running all the time. 

I am then going to be looking at creating different vApps for various different workloads. For example I will have a vApp containing all virtual machines required for a VMware View Configuration where I can quickly power on and it would be part of the same domain/network. 

Some of the vamp’s I am looking to put in place are as follows: 

VMware View 
VMware SRM 
VMware vCloud Director 
Microsoft System Centre 2012 
Microsoft Exchange 
Microsoft SQL
